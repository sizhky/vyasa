# Triage

Decide the feature home first:

- Content syntax: markdown parser/rendering, wikilinks, callouts, code includes, tables.
- Layout/theme config: shell, sidebars, navbar, footer, CSS variables, theme presets.
- Extension: routes, assets, browser behavior, diagrams, task graphs, auth-facing features.

Before proposing syntax, read the matching topic drawer.
Before changing code, cite the concrete file that owns current behavior.
For extension work, read `routes/extensions.md` before editing.
For recent markdown, diagram, item graph, or slide work, read the narrow drawer first.
