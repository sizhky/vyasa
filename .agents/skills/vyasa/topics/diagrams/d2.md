# D2

Use fenced D2 blocks for architecture diagrams, tables, grouped nodes, sequence diagrams, and composition animation.
Requires `d2` installed on the system.

## Basic

````markdown
```d2
---
title: Request Flow
width: 70vw
layout: elk
---
direction: right
user -> api -> db
```
````

## Frontmatter

- `width`: container width, default `65vw`
- `height`: container height, default `auto`
- `title`: fullscreen title or caption
- `layout`: `elk` or `dagre`, default `elk`
- `theme_id`: light D2 theme id
- `dark_theme_id`: dark D2 theme id
- `sketch`: hand-drawn style boolean
- `animate_interval`: ms per animation step
- `animate`: boolean shorthand for animation

## Icons

```d2
web: {
  label: Web App
  icon: "https://api.iconify.design/mdi:web.svg"
}
api: { label: API Server }
web -> api: HTTP
```

Icon sources:

- Iconify API: `https://api.iconify.design/{set}:{name}.svg`
- D2 catalog: `https://icons.terrastruct.com/`

## Structures

Use normal D2 syntax for grouped nodes, SQL tables, and sequence diagrams.

```d2
users: {
  shape: sql_table
  id: int {constraint: primary_key}
  email: string {constraint: unique}
}
orders.user_id -> users.id
```

## Animation

When `animate_interval` is set and no target is specified, Vyasa auto-targets all boards (`*`).
Keep scenario deltas small and labels short.

## Guardrails

- Prefer concise labels and explicit grouping.
- Use supported frontmatter controls only.
- Use `layout: elk` for complex graphs and `dagre` for simpler flows.
- Split dense diagrams into multiple blocks linked by headings.
- For shared interaction behavior, read `topics/diagrams/interactions.md`.
