Vyasa

A lightweight, elegant blogging platform built with FastHTML that renders Markdown files into beautiful web pages with advanced features.

    Router->>Renderer: Parse & render markdown

Installation

From PyPI (recommended)

pip install vyasa

From source

git clone https://github.com/yeshwanth/vyasa.git
cd vyasa
pip install -e .

Quick Start

  1. Create a directory with your markdown files:

    mkdir my-blog
cd my-blog
echo "# Hello World" > hello.md

  2. Run Vyasa:

    vyasa .

  3. Open your browser at http://127.0.0.1:5001

Configuration

Vyasa supports three ways to configure your blog (in priority order):

  1. .vyasa configuration file (TOML format)
  2. Environment variables
  3. Default values

Using a .vyasa Configuration File

Create a .vyasa file in your blog directory or current directory:

# Blog title (default: derives from root folder name)
title = "My Awesome Blog"

# Root folder containing markdown files (default: current directory)
root = "."

# Server host (default: 127.0.0.1)
# Use "0.0.0.0" to make the server accessible from network
host = "127.0.0.1"

# Server port (default: 5001)
port = 5001

All settings in the .vyasa file are optional. See .vyasa.example for a full example.

Environment Variables

You can also use environment variables as a fallback:

  • VYASA_ROOT: Path to your markdown files (default: current directory)
  • VYASA_TITLE: Your blog's title (default: folder name)
  • VYASA_HOST: Server host (default: 127.0.0.1)
  • VYASA_PORT: Server port (default: 5001)

Examples

Using a .vyasa file:

# Create a .vyasa file in your blog directory
cd /path/to/your/blog
cat > .vyasa << EOF
title = "My Tech Blog"
port = 8000
EOF

vyasa

Using environment variables:

export VYASA_ROOT=/path/to/your/markdown/files
export VYASA_TITLE="My Awesome Blog"
export VYASA_PORT=8000
vyasa

Passing directory as argument:

vyasa /path/to/your/markdown/files

Configuration priority example:

If you have both a .vyasa file with port = 8000 and an environment variable VYASA_PORT=9000, the .vyasa file takes precedence and port 8000 will be used.
